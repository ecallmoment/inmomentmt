Projects

  • Upload

    • - Click 'Add Project' from the 'Projects' page.
    • - Type in a new project name.
    • - Choose a CSV tag map file. Each row should include the whole hierarchy for the tag.
    • - Choose a CSV file of 'text analyzer' output. This is essentially the outputted Excel file, but with the headings to the side deleted, and saved as a CSV.
    • - Click 'Save.'

  • View

    • - 'Tagging' brings you to the tagging page for that project.
    • - 'Results' brings you to the results page for that project.
    • - The trashcan deletes that project from the database. (There's a check to make sure you want to do that.)

Tagging

  • Autotagged Comments

    • - You will start from the beginning (if you're just starting) or from the place you left off.
    • - Use 'Back' and 'Forward' to navigate without changing the classification of the comment.
    • - 'Yes' means that the highlighted region is a representation of the bolded tag.
    • - 'No' means that the highlighted region is not a representation of the bolded tag. This tag—unhighlighted—will be added to the comments to manually tag.
    • - 'Flag This' allows you to flag the tag if you find it particularly problematic for whatever reason.
    • - Switch to Manual Tagging at any time.
    • - You can change the classification of any comment/tag at any time. The most recent classification will be what's saved in the database.

  • Manual Tagged Comments

    • - You will start from the beginning (if you're just starting) or from the place you left off.
    • - Use 'Back' and 'Next' to navigate without changing the classification of the comment.
    • - Select as many tags as you want from the menu (click a second time to remove the tag). The tags will apply to the entire shown comment by default.
    • - If you want to only apply a tag to a certain section of the comment, highlight the selection, and then apply the tags to that selection. This newly created comment/tag pair will go to the end of the comments in the manual tagging section.

Results

  • View

    • - Each category is sorted by tag name.
    • - Highlighting provides a quick glance at the text covered by the tag.

  • Download

    • - Downloads a CSV of the data shown on the 'Results' page.
    • - Columns include: comment number, the result, the tag name, the covered text, and the full comment.
    • - AT Correct means that the autotag was correct, while AT Incorrect means that the autotag was incorrect and may show up in a MT CREATED result, which is the result of manual tagging and additional highlighting. AT Flagged refers to the comment being flagged during the auto-tag checking process.
